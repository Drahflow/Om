Om
A Simple Programming Language
Using

Om can be used as:

  • A C++ library.
  • A stand-alone interpreter, built from a script-generated source project.

Dependencies

The code depends on the following libraries:

  • ICU (ICU4C 50.1.2 or greater)
  • Boost (1.53.0 or greater)

To run the scripts which build the above libraries and generate the project, the following programs are required:

  • CMake (2.8.10 or greater)
  • Cygwin (with bash, GNU make, ar, and ranlib) (Windows only)

To build the documentation target in the project, the following programs are required:

  • Doxygen (1.8.3.1 or greater)
  • Graphviz (2.30.1 or greater)

To ensure that the correct applications are used, the command line path should be in the following order:

  • Graphviz, Doxygen, and CMake
  • Cygwin ("[cygwin]/bin") (Windows only)
  • Any other paths

Library

Om is a header-only C++ library that can be incorporated into any C++ or Objective-C++ project as follows:

  • Add the Om "code" directory to the include path and include the desired files. Inclusion of any operation header files will automatically add the corresponding operation to the global system. Include "om.hpp" to include all Om header files.
  • Configure the project to link to the code dependencies as necessary, built with the correct configuration for the project. See "premake.sh" and "premake.bat" for guidance.
  • Call the Om::System::Initialize function prior to use (e.g. in the main function), passing in the desired UTF-8 locale string (e.g. "en_US.UTF-8").
  • Construct an Om::Environment, populate with any additional operator-program mappings, and call one of its Om::Environment::Evaluate functions to evaluate a program.

For more in-depth usage of the library, see the Om code documentation.

Project

Scripts are provided for generating a source project that contains targets for building the interpreter, unit tests, and documentation.

Platforms

Xcode

The following scripts have been tested with Xcode 4.6 on Mac OS X 10.8.2 (Mountain Lion):

  • "premake.sh": Builds the external library dependencies as required for Om. Takes the following arguments (in order):
    • The ICU root directory path. To create this directory, download and unpack the ICU4C 50.1.2 code archive (Unix) from the ICU download page. Note: due to a limitation of the ICU configuration script, the path cannot contain spaces.
    • The Boost root directory path. To create this directory, download and unpack the Boost 1.53.0 code archive (Unix) from the Boost download page. Note: due to a limitation of the Boost configuration script, the path cannot contain spaces.
  • "make.sh": Creates the "Om.xcodeproj" file in "products/projects/xcode". Takes the same arguments as "premake.sh".
  • "remake.sh": Updates the Xcode project to reflect changes to the code directory tree. Takes no arguments.
  • "unmake.sh": Removes the directory generated by "make.sh". Takes no arguments.

Visual Studio

The following scripts have been tested with Visual Studio 2010 Express on Windows Vista:

  • "premake.bat": Builds the external library dependencies as required for Om. Takes the following arguments (in order):
    • The ICU root directory path. To create this directory, download and unpack the ICU4C 50.1.2 code archive (Unix, to work best with Cygwin) from the ICU download page.
    • The Boost root directory path. To create this directory, download and unpack the Boost 1.53.0 code archive (Windows) from the Boost download page.
  • "make.bat": Creates the "Om.sln" file in "products/projects/vs". Takes the same arguments as "premake.bat".
  • "remake.bat": Updates the Visual Studio solution to reflect changes to the code directory tree. Takes no arguments.
  • "unmake.bat": Removes the directory generated by "make.bat". Takes no arguments.

Other

Because Om is written in portable C++, it should be possible to build the code on any mainstream platform by adapting the scripts and "CMakeLists.txt" file. Please share any enhancements to the build process and/or report any code portability issues encountered.

Targets

Om

The Om target builds the Om interpreter executable into "products/projects/[platform]". The Om interpreter:

  • Accepts an optional command-line argument that specifies the desired UTF-8 locale string. The default value is "en_US.UTF-8".
  • Reads input from the standard input stream, ending at the first unbalanced end brace, and writes output to the standard output stream as it is computed.

Om.Test

The Om.Test target builds the Om.Test test executable, which runs all unit tests, into "products/projects/[platform]".

Om.Documentation

The Om.Documentation target builds this documentation into the following folders in "products/documentation/doxygen":

  • "html": The HTML documentation. To view in a browser, open "index.html".
  • "xml": The XML documentation, which can be read by an integrated development environment to show context-sensitive documentation.
