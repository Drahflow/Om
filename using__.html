Using

Om can be used as a library or a stand-alone interpreter.

Library

Om is a header-only C++ library that can be incorporated into any C++ or Objective-C++ project as follows:

  • Add the Om code directory to the include path and include the desired files. Inclusion of any operation files will automatically add the corresponding operation to the global system. Include om.hpp to include all Om header files.
  • Configure the project to link to the required ICU (ICU4C 49.1.2) and/or Boost (1.51.0) library dependencies, and build these with the correct configuration for the project. See make.sh and make.bat for guidance.
  • Call the Om::System::Initialize function prior to use (eg. in the main function), passing in the desired UTF-8 locale (eg. "en_US.UTF-8").
  • Construct an Om::Environment, populate with any additional operator-program mappings, and call one of its Evaluate functions to evaluate a program.

Scripts

Scripts are included that use CMake to generate project files with the following targets:

  • Om:
    • Builds the Om executable into products/projects/[platform]. The Om executable is an interpreter that:
      • Accepts an optional argument that specifies the desired UTF-8 locale. The default value is "en_US.UTF-8".
      • Runs all unit tests (if compiled with Om_Macros_Test_ defined).
      • Reads input from the standard input stream, ending at the first unbalanced end brace.
      • Writes output to the standard output stream.
  • Om Documentation:
    • Builds the Om documentation into products/documentation/doxygen. To view the HTML documentation in a browser, open html/index.html.

Xcode

To generate the Om.xcodeproj file (in products/projects/xcode):

  • Ensure that Doxygen and Graphviz (via MacPorts) are installed and available to applications via the path variable. This article explains how to do so.
  • Run make.sh and pass it the following arguments (in order):
    • The ICU root directory path. To create this directory, download and unpack the ICU4C 50.1.2 code archive from the ICU download page.
    • The Boost root directory path. To create this directory, download and unpack the Boost 1.52.0 code archive from the Boost download page.
    • Optional: The UnitTest++ root directory path. If omitted, unit tests will be excluded from the build. To create this directory, download and unpack the UnitTest++ 1.4 code archive from the UnitTest++ download page.

To update the Xcode project to reflect changes to the code directory tree:

  • Run remake.sh.

To remove the directory generated by make.sh:

  • Run unmake.sh.

The above scripts have been tested with Xcode 4.6 on Mac OS X 10.8.2 (Mountain Lion).

Visual Studio

To generate the Om.sln file (in products/projects/vs):

  • Ensure that Doxygen and Graphviz are installed and in the path variable.
  • Run make.bat and pass it the following arguments (in order):
    • The ICU root directory path. To create this directory, download and unpack the ICU4C 49.1.2 code archive from the ICU download page. (Note that ICU4C 50.1.1 cannot be used due to a bug that prevents building on Windows.)
    • The Boost root directory path. To create this directory, download and unpack the Boost 1.52.0 code archive from the Boost download page.
    • Optional: The UnitTest++ root directory path. If omitted, unit tests will be excluded from the build. To create this directory, get the UnitTest++ 1.4 code via Subversion checkout (described on the UnitTest++ developer page) and apply the patch attached to the Visual Studio 2010 support ticket. The UnitTest++ root directory is the UnitTest++ subdirectory of the Subversion checkout directory.

To update the Visual Studio solution to reflect changes to the code directory tree:

  • Run remake.bat.

To remove the directory generated by make.bat:

  • Run unmake.bat.

The above scripts have been tested with Visual Sutdio 2010 Express on Windows Vista.

Other Platforms

Because Om is written in portable C++, it should be possible to build the code on any mainstream platform by adapting the scripts and CMakeLists.txt file. If this is of interest, please contribute the result via pull request and/or report any code portability errors encountered.
