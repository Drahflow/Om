Om
A Simple Programming Language
Using the Code

The Om source code can be used for:

  • Building a stand-alone interpreter from a script-generated source project.
  • Including as a C++ header-only library.

Platforms

Mac OS X and Ubuntu

Scripts ending in ".sh" are provided for use on Mac OS X 10.8.3 and Ubuntu 12.0.4.

Note: The ICU4C and Boost build scripts for Unix do not support paths containing spaces.

Windows

Scripts ending in ".bat" are provided for use on Windows (Vista, 7, 8), and must be run from the Visual Studio command prompt.

Other

Because Om is implemented with portability in mind, it should be possible to build the code on any mainstream platform with minor adaptation of the scripts and "CMakeLists.txt" files. Please share any enhancements to the build process and/or report any code portability issues encountered.

Dependencies

Programs

To run scripts which build the dependency libraries and generate the project, the following programs are required:

  • CMake (2.8.7 or greater)
  • Mac OS X:
    • Xcode (4.6 or later)
  • Windows:
    • Visual Studio (2010 or later)
    • Cygwin (with bash, GNU make, ar, and ranlib)
  • Ubuntu:
    • Build-Essential package (sudo apt-get install build-essential)

To build the documentation target in the project, the following additional programs are required:

  • Doxygen (1.8.4 or greater; note that earlier versions may generate warnings)
  • Graphviz (2.26.3 or greater)

To ensure that correct programs are used, programs should be listed in the command line path in the following order:

  • Graphviz, Doxygen, and CMake
  • Cygwin ("[cygwin]/bin") (Windows only)
  • Any other paths

Libraries

ICU4C

Om depends on ICU4C (51.2 or greater), the C++ implementation of the ICU library. ICU4C 51.2 can be downloaded and built into "input/dependencies/Icu4c/output/artifacts" by navigating to the "input/dependencies/Icu4c" directory and running the "generate" script.

Boost

Om depends on the Boost library (1.53.0 or greater). Boost 1.53.0 can be downloaded and built into "input/dependencies/Boost/output/artifacts" by navigating to the "input/dependencies/Boost" directory and running the "generate" script.

Project

A source project, containing targets for building the interpreter, tests, and documentation, can be built into "output/artifacts" by running the "generate" script, with the CMake generator as an argument, in the root directory. By default, this script automatically downloads and builds all external dependency libraries, but this behaviour can be overridden by passing paths of pre-installed dependency libraries to the script:

  • -D Icu:Path="[absolute path]", where [absolute path] is the absolute path of the ICU4C install directory.
  • -D Boost:Path="[absolute path]", where [absolute path] provides the absolute path of the Boost install directory.

Targets

Om.Interpreter

The Om.Interpreter target builds the interpreter executable as "output/artifacts/[generator]/[configuration]/Om.Interpreter". See Using the Interpreter for instructions on how to use the interpreter.

Om.Test

The Om.Test target builds the test executable, which runs all unit tests, as "output/artifacts/[generator]/[configuration]/Om.Test".

Om.Documentation

The Om.Documentation target builds this documentation into the following folders in "output/artifacts/Doxygen":

  • "html": The HTML documentation. To view in a browser, open "index.html".
  • "xml": The XML documentation, which can be read by an integrated development environment to show context-sensitive documentation.

Library

Om is a header-only C++ library that can be incorporated into any C++ or Objective-C++ project as follows:

  • Add the Om "input/code" directory to the include path and include the desired files. Inclusion of any operation header files will automatically add the corresponding operation to the global system. Include "om.hpp" to include all Om header files.
  • Configure the project to link to the code dependencies as necessary, built with the correct configuration for the project. See the dependency "generate" scripts for guidance.
  • Call the Om::System::Initialize function prior to use (e.g. in the main function), passing in the desired UTF-8 locale string (e.g. "en_US.UTF-8").
  • Construct an Om::Environment, populate with any additional operator-program mappings, and call one of its Om::Environment::Evaluate functions to evaluate a program.

For more in-depth usage of the library, see the Om code documentation.
